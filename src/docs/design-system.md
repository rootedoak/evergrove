Evergrove Design Principles
1. Mobile First

Every screen should feel native on a phone. Desktop is an enhancement, not the primary experience.

2. One Primary Action

Each screen should have one obvious thing the user can do next.

3. Information Hierarchy

Show what matters today first. Historical information belongs deeper in the experience.

4. Calm Interfaces

Prefer whitespace, typography, and grouping over borders, boxes, and visual noise.

5. Reusable Components

If you build the same UI twice, it should become a component.

6. Story Over Statistics

Whenever possible, tell the user what's happening instead of just showing numbers.

Example:

❌ "3 Tasks"

✅ "Three tasks need your attention today."

7. Progressive Disclosure

Don't show advanced options until they're needed.

8. Family First

The app should always make it obvious:

Who did it
Who owns it
Who it affects

9. Assistant Everywhere

The Assistant isn't a page—it's a capability that appears naturally wherever it can help.

10. Delight Through Polish

Micro-interactions, smooth transitions, subtle animations, and thoughtful empty states should make the app feel alive without getting in the way.
